01: Tổng quan — Giới thiệu bài toán

BỐI CẢNH & GIẢI PHÁP ĐỀ XUẤT

Bối cảnh

Trong môi trường an ninh mạng hiện đại, mã độc ngày càng tinh vi với các kỹ thuật ngụy trang và biến đổi cấu trúc (đa hình).

Các phương pháp truyền thống bộc lộ điểm yếu kép:

  • Quét bằng mã băm (Hash) hoặc dựa vào phần mở rộng tệp tin (.exe, .doc) có tốc độ nhanh nhưng dễ dàng bị tin tặc qua mặt.
  • Ngược lại, phương pháp quét sâu (Deep Scan) vào nội dung dữ liệu thô mang lại độ chính xác cao nhưng lại gây cạn kiệt tài nguyên và thời gian xử lý của hệ thống.

Giải pháp đề xuất

  • Hệ thống sử dụng Python kết hợp với thư viện YARA để phát hiện mã độc bằng kỹ thuật so khớp mẫu.
  • Tích hợp thành một hệ thống tự động có khả năng rà quét các tệp tin đơn lẻ và đệ quy toàn bộ thư mục đối với các vùng dữ liệu đáng ngờ.
02 — Công cụ phân tích

TỔNG QUAN VỀ CÔNG CỤ YARA

Khái niệm

YARA là gì?

  • YARA là công cụ mã nguồn mở, dùng để tạo quy tắc mô tả mã độc dựa trên chuỗi văn bản hoặc mẫu nhị phân.
  • Hiện do VirusTotal bảo trợ và phát triển liên tục.
Cơ chế hoạt động

Khớp dấu hiệu trên dữ liệu thô

YARA phân loại và nhận diện mã độc dựa trên các dấu hiệu nhận dạng (Signature-based matching). Công cụ hoạt động bằng cách quét và so sánh dữ liệu thô của tệp tin với các mô tả logic do người dùng định nghĩa: bao gồm chuỗi văn bản, mẫu nhị phân Hex hoặc biểu thức chính quy.

03 — Cấu trúc

CẤU TRÚC MỘT TẬP LUẬT YARA

Một quy tắc (Rule) YARA tiêu chuẩn được chia thành 3 thành phần cốt lõi:

Meta (Siêu dữ liệu)

Lưu trữ thông tin về tác giả, mô tả và ngày tạo của quy tắc.

Strings (Chuỗi dấu hiệu)

Định nghĩa các chuỗi văn bản (Text), chuỗi thập lục phân (Hex) hoặc biểu thức chính quy (Regex).

Condition (Điều kiện logic)

Các phép toán logic quyết định khi nào quy tắc này được kích hoạt.

Ví dụ minh họa cấu trúc cơ bản:

rule MalwareExample { meta: author = "Hồ Quốc Long — Nguyễn Bá Thiều Khôi Nguyên" description = "Phát hiện đoạn mã độc cụ thể" strings: $s1 = "malicious_code" condition: $s1 }
04 — YARA Rules

VÍ DỤ: RULE PHÁT HIỆN WANNACRY

import "pe" rule Ransomware_WannaCry { meta: author = "Nguyễn Bá Thiều Khôi Nguyên - Hồ Quốc Long" description = "Phát hiện mã độc tống tiền WannaCry dựa trên các chuỗi đặc trưng và hành vi." reference_1 = "https://www.cisa.gov/news-events/alerts/2017/05/12/indicators-associated-wannacry-ransomware" reference_2 = "https://attack.mitre.org/software/S0366/" date = "2026-03-18" malware_family = "WannaCry" severity = "high" strings: // Tên miền kill-switch xuất hiện trong các biến thể WannaCry ban đầu. $killswitch = "iuqerfsodp9ifjaposdfjhgosurijfaewrwergwea.com" ascii nocase // Dùng tiền tố $ioc_ cho tất cả các dấu vết còn lại để dễ nhóm trong condition $ioc_svc_name = "mssecsvc2.0" ascii nocase $ioc_task_binary = "tasksche.exe" ascii nocase $ioc_decryptor = "@WanaDecryptor@.exe" ascii nocase $ioc_task_dl = "taskdl.exe" ascii nocase $ioc_readme = "@Please_Read_Me@.txt" ascii nocase $ioc_ransom_title = "Wanna Decryptor" ascii wide nocase $ioc_ransom_text = "Oops, your files have been encrypted!" ascii wide nocase $ioc_ext_wncry = ".WNCRY" ascii nocase $ioc_ext_wnry = ".WNRY" ascii nocase condition: pe.is_pe and ( // Tín hiệu mạnh: có kill-switch và ít nhất 1 dấu vết khác. ($killswitch and 1 of ($ioc_*)) or // Trường hợp biến thể đã bị vá kill-switch: cần ít nhất 3 dấu vết. (3 of ($ioc_*)) ) }
05 — Tích hợp công cụ

TÍCH HỢP YARA VÀO MÔI TRƯỜNG PYTHON

Gói thư viện: Sử dụng yara-python, được cài đặt qua trình quản lý gói pip của Python.

Biên dịch (Compile)

Chuyển đổi tệp luật văn bản (.yar) thành đối tượng thực thi trong bộ nhớ (RAM) nhằm tối ưu hóa tốc độ xử lý.

So khớp (Match)

Áp dụng đối tượng biên dịch lên tệp tin hoặc luồng dữ liệu thô để phát hiện vi phạm.

Mã nguồn minh họa:

import yara # Bước 1: Biên dịch tập luật vào bộ nhớ rules = yara.compile('rules/index.yar') # Bước 2: Rà quét tệp tin khả nghi matches = rules.match('suspicious.exe') print(matches) # Trả về danh sách quy tắc vi phạm
06 — Kiến trúc tổng thể

TỔNG QUAN KIẾN TRÚC

Mô hình nền tảng:

  • Hệ thống phát hiện mã độc được thiết kế và vận hành hoàn toàn trên ngôn ngữ Python.
  • Tích hợp sâu thư viện YARA làm trung tâm của động cơ rà quét tệp tin dựa trên bộ rule thiết lập sẵn.

Thiết kế kiến trúc phân tầng:

Tầng Giao diện người dùng

User Interface

Tiếp nhận tệp tin hoặc thư mục đầu vào từ người dùng và trình bày kết quả rà quét trực quan.

Tầng Xử lý ứng dụng

Application Layer

Đảm nhiệm việc đọc dữ liệu, biên dịch rule và thực thi cơ chế rà quét hai lớp: Đối chiếu mã băm qua cơ sở dữ liệu -> phân tích đặc trưng bằng YARA.

Tầng Lưu trữ dữ liệu

Data Layer

Ghi nhận kết quả rà quét, lưu trữ lịch sử hệ thống và các siêu dữ liệu (metadata) liên quan đến tệp tin khả nghi.

07 — Cơ chế xử lý tệp nén

CƠ CHẾ XỬ LÝ TỆP NÉN (ZIP/RAR)

Vấn đề

Một số mã độc được lưu trữ trong các tệp nén (ZIP/RAR), làm xáo trộn cấu trúc nhị phân để qua mặt cơ chế quét YARA.

Giải pháp

Cơ chế Vùng cách ly

  • Nhận diện tệp nén qua thư viện zipfile của Python.
  • Bung nén đệ quy toàn bộ nội dung tệp tin vào vùng cách ly.
  • Chuyển hướng quét YARA vào từng tệp con.
  • Dọn dẹp tự động vùng cách ly sau hoàn tất để tiết kiệm bộ nhớ.

09 — Archive

Cơ chế quét Archive

Đặc điểm

  • Hỗ trợ ZIP và 7z
  • Extract vào memory — không ghi ra disk
  • Quét từng file bằng YARA rules
  • Hỗ trợ nested archives (depth ≤ 3)

Bảo vệ

Archive BombRatio ≤ 100:1
Nested DepthTối đa 3 lớp
File CountTối đa 1000
Size LimitTối đa 100MB
10 — Database

PostgreSQL Database

Cơ sở dữ liệu: yara_malware_signatures

Bảng malware_signatures

  • Lưu malware signatures từ API
  • SHA256 hash (UNIQUE)
  • MD5, SHA1, SHA3-384 hashes
  • Index cho truy vấn nhanh

Bảng scan_results

  • Lịch sử quét
  • Detection method: HASH_MATCH, YARA_MATCH, CLEAN
  • Timestamp tự động

Nguồn dữ liệu: MalwareBazaar API (abuse.ch) — lấy signatures cho 14 họ malware
